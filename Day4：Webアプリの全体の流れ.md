🟦 Day4 振り返り：Webアプリの全体の流れ

今日やったこと・学習したこと

Webアプリは
React → Laravel → DB → Laravel → React
という「往復構造」で動いていることを学んだ。
Reactは画面担当、Laravelは処理担当、DBは保存担当で、
画面更新はデータ（state）が変わった結果として起きる。


理解できた重要ポイント

画面を直接更新するのではなく、
データを更新することが本体であり、
表示はその結果でしかないという考え方。


疑問に思ったこと・結論

「Reactが自動で画面更新する」と感じていたが、
正確には Laravelから返ったデータでstateを更新した結果、再描画されている と理解できた。
