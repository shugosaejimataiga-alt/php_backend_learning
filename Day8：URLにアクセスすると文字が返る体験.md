🟦 Day8：URLにアクセスすると文字が返る体験


今日やったこと

Ubuntu（WSL）で Laravel プロジェクトを起動した
→ ./vendor/bin/sail up

ブラウザで http://localhost にアクセスし
Laravelがローカルで動いていることを確認した

routes/web.php を編集し、
自分で書いた文字を画面に表示させた



学習したこと（核心）

バックエンドの基本は
**「URLに来たリクエストに対して、何かを返す」**こと

Laravelではそれを ルーティングで定義する


Route::get('/', function () {
    return 'Day8 OK';
});


この1行は、

「/ に GET でアクセスされたら、Day8 OK を返せ」

という 約束 を書いている。



各要素の意味

Route::get
→ GET通信（見る・取得する） 用の入口

'/'
→ トップページのURL

function () { ... }
→ アクセスされた瞬間に実行される処理

return 'Day8 OK';
→ ブラウザに返す内容（今回は文字）



理解した重要なポイント

GET は HTTP通信の種類（HTTPメソッド）

見る・取得するための通信

URL と HTTPメソッドの組み合わせで
「このURLは何をする場所か」 が決まる

バックエンドはまず
返すことができて初めて存在意義を持つ



今日出た疑問と答え

Q. get って何？
→ A. HTTPメソッドの一種で、「見る・取得する」通信

Q. 'Hello Laravel' に変えたら？
→ A. その文字がそのまま画面に表示される



Day8の結論（最重要）

「URLにアクセスすると、Laravelが自分の書いた内容を返す」
というバックエンドの入口を完全に体験した。
