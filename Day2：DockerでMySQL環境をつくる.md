今日は「MySQL を動かすための土台をつくる日」です。
コードを書くというより 環境構築の本質を理解する日 でした。

■ まず最初に押さえる超シンプルな話

MySQL はそのままでは Windows では動きません。
Docker（コンテナの箱）を用意して、その中で動かします。

だから今日やったことは、

MySQL が安全・安定して動く “部屋（コンテナ）” を作った

ということ。

この作業は IT エンジニアとして、
今後のプロジェクトでもほぼ確実に出てきます。



■ 今日の本質的に大切な考え方

✔ 1) Docker は「設定ファイルのある場所で動かす」

Docker は
docker-compose.yml が置いてあるフォルダだけを読み取って動く。

だから、
まず cd mysql-docker でそのフォルダに移動する必要があった。

これは“ファイルのあるキッチンで料理する” みたいなもの。



✔ 2) Windows でも Docker が動くのは「裏に Linux がいるから」

Windows はそのままでは Docker を動かせない。
だからインストール時にあなたが許可した WSL2（軽量Linux） が裏側で動いている。

あなたが Linux を意識する必要はほぼないが、
Docker が動く仕組みとして知っておくのはとても良い。

Windows
 ↓
WSL2（Linux）
 ↓
Docker Engine
 ↓
MySQL コンテナ

という構造になっている。



✔ 3) パスワード入力は「何も見えなくて正常」

ターミナルのパスワードは、

入力しても表示されない
何文字打ったかも表示されない
Backspace も見えない

これはセキュリティのためで 完全に正常な挙動。

最後に正しく root と打って Enter すればログインできる。



✔ 4) exit; で抜けても MySQL は止まらない

exit; は “MySQL の中から出る” だけ。

コンテナは動いたまま
MySQL も動いたまま

またログインすればすぐ再開できる

今日の作業をやり直す必要はない。



■ 今日の作業としてやったこと

✔ mysql-docker フォルダを作成
✔ docker-compose.yml を作成して中身を貼る
✔ docker compose up -d で MySQL を起動
✔ docker ps で動いていることを確認
✔ docker exec … で MySQL の中にログイン
✔ パスワード入力の仕組みを理解
✔ exit; で安全に抜けられることを確認
✔ Linux / Docker / Windows の関係も理解
